# Vergleich der führenden Cloud-Anbieter

Der Markt für Cloud-Computing wird von den drei "Hyperscalern" dominiert: **Amazon Web Services (AWS)**, **Microsoft Azure** und **Google Cloud Platform (GCP)**. Obwohl alle drei Anbieter eine globale Infrastruktur für nahezu jeden Anwendungsfall bereitstellen, unterscheiden sie sich stark in ihrer Philosophie, ihren Kernkompetenzen und ihren Ökosystemen.

---

## 1. Die wichtigsten Dienstleistungen im Vergleich

Die Anbieter nutzen unterschiedliche Bezeichnungen für technologisch vergleichbare Dienste. 

| Dienstleistungskategorie | AWS | Microsoft Azure | Google Cloud (GCP) |
| :--- | :--- | :--- | :--- |
| **Virtuelle Server (Compute)** | EC2 (Elastic Compute Cloud) | Virtual Machines (VMs) | Compute Engine |
| **Verwaltetes Kubernetes** | EKS | AKS | GKE |
| **Speicherlösungen (Objekte)** | S3 (Simple Storage Service) | Blob Storage | Cloud Storage |
| **Datenbanken (Relational)** | RDS / Aurora | Azure SQL Database | Cloud SQL / AlloyDB |
| **Datenbanken (NoSQL)** | DynamoDB | Cosmos DB | Cloud Bigtable / Firestore |
| **AI / ML Services** | Amazon Bedrock / SageMaker | Azure OpenAI Service | Vertex AI / Gemini |
| **Netzwerkdienste** | VPC (Virtual Private Cloud) | Virtual Network (VNet) | VPC (Globales Netzwerk) |

### Besondere Stärken der Anbieter:
* **AWS:** Bietet den mit Abstand größten und reifsten Service-Katalog (über 200 Dienste). Mit *Amazon Bedrock* setzt AWS auf maximale Flexibilität bei der Auswahl verschiedener KI-Modelle.
* **Azure:** Besticht durch die nahtlose Integration mit Microsoft-Produkten (Windows Server, SQL Server, Active Directory, Microsoft 365). Stark bei Enterprise-KI-Lösungen durch die Partnerschaft mit OpenAI.
* **GCP:** Vorreiter bei Open-Source-Technologien (Kubernetes) und Big-Data-Analysen (*BigQuery*). Mit *Vertex AI* extrem stark bei der nativen Entwicklung von KI-Modellen.

## 2. Vergleich relevanter Kriterien

### Kostenstruktur
* **Gemeinsamkeit:** Alle nutzen primär ein "Pay-as-you-go"-Modell sowie Rabatte für langfristige Verpflichtungen.
* **AWS:** Sehr flexibel, aber die Preisstruktur ist komplex. Versteckte Kosten (z. B. für Datenexporte) erfordern ein striktes FinOps-Management.
* **Azure:** Bietet erhebliche Kostenvorteile ("Azure Hybrid Benefit") für Unternehmen, die bereits bestehende Microsoft-Lizenzen besitzen.
* **GCP:** Bietet oft die transparenteste Preisstruktur. Rabatte für kontinuierliche Nutzung (*Sustained Use Discounts*) werden teilweise automatisch gewährt.

### Skalierbarkeit
* Alle drei Anbieter bieten nahezu unbegrenzte horizontale und vertikale Skalierung auf Knopfdruck.
* **GCP** hat im Bereich Container-Skalierung die Nase vorn, da die Google Kubernetes Engine (GKE) extrem tief und performant integriert ist.
* **AWS** bietet dank seiner über 500 verschiedenen Instanztypen (inklusive eigener *Graviton*-Prozessoren) die granularste Kontrolle bei der Skalierung.

### Sicherheit
* Alle drei Plattformen erfüllen die weltweit strengsten Compliance-Standards (C5, ISO 27001, DSGVO) und nutzen das *Shared Responsibility Model*.
* **Azure** ist Marktführer im Bereich Identitätsmanagement dank *Microsoft Entra ID* (früher Active Directory), was hybride Umgebungen optimal absichert.
* **GCP** schützt Daten standardmäßig im Ruhezustand und bei der Übertragung und bietet starke Lösungen für *Confidential Computing*.

### Verfügbarkeit und Zuverlässigkeit
* Die Anbieter garantieren über SLAs meist eine Verfügbarkeit von 99,9 % bis 99,99 % für Kerndienste.
* **AWS** verfügt über das ausgereifteste globale Netz aus Regionen und isolierten Verfügbarkeitszonen (Availability Zones).
* **GCP** hebt sich durch sein privates, weltweites Glasfasernetzwerk ab, was global für sehr konstante, niedrige Latenzen sorgt.

### Kundensupport
* Alle bieten gestaffelte Support-Modelle (vom kostenlosen Basis-Support bis zum 24/7 Enterprise-Support).
* **AWS** besitzt das größte Partner-Ökosystem und die größte Entwickler-Community.
* **Azure** punktet bei klassischen Großkunden, da Cloud-Support oft in bestehende Enterprise-Verträge integriert werden kann.

---

## Fazit: Welcher Anbieter für wen?

* Wählen Sie **AWS** für maximale Flexibilität, die größte Auswahl an Diensten und ein riesiges Ökosystem.
* Wählen Sie **Azure**, wenn Ihr Unternehmen tief im Microsoft-Ökosystem verwurzelt ist und Sie hybride Cloud-Lösungen suchen.
* Wählen Sie **Google Cloud**, wenn Ihr Fokus auf moderner Container-Entwicklung, Big Data oder KI-Modellierung liegt.
